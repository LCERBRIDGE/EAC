Client Commands

The following command set is recognized by the EAC, and currently can be sent to the EAC from a controlling client.

The commands are designed for asynchronous communication. There is a prompt response that acknowledges the command or rejects it, though other messages may intervene between the command and the response. In some cases, there is a delayed response when the requested action has been completed or has failed.

Responses always begin by echoing the command to which they are responding.

Currently, versions 3.x (version 3.4 being the latest) work on the 70-m antennas. Version 4.0and later versions work at DSS-13. Commands which are known to work reliably under both current series are colored green.

Warning: Commands which work under version 3.x but are not listed as version 4.0+ commands will probably be accepted but may behave unpredictably. When such behaviour is known, it is described below.

Version 6.2 is intended to be the first version to consolidate the two series.

Receiver Commands

The following commands are not processed by the EAC, except to verify that they are valid. The commands are processed by scripts defined in the rac_ctlrc or rac_monrcstart-up file for the Tcl interpreter assigned to the EAC upon connecting as a client of racsrv. The script directory on the RAC is either /usr/local/lib/racsrv.dflt or /usr/local/lib/racsrv.PROJECT, where PROJECT is an optional command-line parameter specified when racsrv is started.

Any of these commands may fail with
command: rejected. RAC not connected.

CALOFF
Project dflt 70-m

Responses
caloff: rejected. If an attempt to turn on an L-band noise diode fails.
caloff: completed. If the noise diode was turned on succesfully, or if an S or X-band front end is selected, in which case nothing was done.
